JAVA中Hashmap实现

发表于 2020-06-09 |

HashMap实现实现了所有Map的函数，允许Null作为key和value。但是HashMap中不保证值的插入顺序。

Hashmap中get和put操作的在hash函数能分散key到所有桶上时是常数时间的。

Hashmap通过两个变量控制其性能：initital capacity和load factor。当Hashmap的长度大于二者的乘积时，Hashmap reszie，使桶的数量为原来的二倍。

Load factor的初始值为0.75,这个值越大，空间利用率越高，但是会增加查找时间。如果在创建HashMap时提供对所需空间的估计，指定initital capacity,可以减少自动resize花费的时间。

HashMap并不是线程安全的，如果多个线程同时访问该Hashmap，其中有一个线程更改了Hashmap的结果，那么必须对Hashmap使用同步机制。修改结构指的是增加或减少mapping，只更新某个key对应的value不算修改结构。

Hashmap使用数据，链表，红黑树实现。利用hash函数计算Node在数组中的位置，当两个Nodehash相同时用链表连接，当hashmap中的节点过多时，将数组中的链表转为红黑树，以提高查找效率。

由于TreeNode的大小是Node的两倍，因此一般情况下不适用红黑树结构，只有当一个桶内存储了太多的节点(阈值由TREEIFY_THRESHOLD指定)才变为树结构，当存储的节点变少时，会退化为链表结构。

一般来说，在load factor为0.75以及随机hashcode的情况下，桶内存放节点数量的频率服从参数为0.5的泊松分布，也即hash碰撞的概率是比较低的。

在HashMap中，哈希桶数组table的长度length大小必须为2的n次方(一定是合数)，这是一种非常规的设计，常规的设计是把桶的大小设计为素数。相对来说素数导致冲突的概率要小于合数，具体证明可以参考http://blog.csdn.net/liuqiyao_01/article/details/14475159，Hashtable初始化桶大小为11，就是桶大小设计为素数的应用（Hashtable扩容后不能保证还是素数）。HashMap采用这种非常规设计，主要是为了在取模和扩容时做优化，同时为了减少冲突，HashMap定位哈希桶索引位置时，也加入了高位参与运算的过程。

扩容方法

resize方法用于初始化Hashmap或者将其容量扩充为原来的两倍。

由于hashmap的长度始终为2的幂，因此，扩容为原来的二倍相当于多了一位掩码用于决定node的位置，因此每个节点要么位于原来的位置，要么移动2的幂（由掩码位置的值决定）。

final Node<K,V>[] resize() {
    Node<K,V>[] oldTab = table;
    int oldCap = (oldTab == null) ? 0 : oldTab.length;
    int oldThr = threshold;
    int newCap, newThr = 0;
    if (oldCap > 0) {
        // 如果原来的容量超出了最大的大小，则扩充threshold
        if (oldCap >= MAXIMUM_CAPACITY) {
            threshold = Integer.MAX_VALUE;
            return oldTab;
        }
        else if ((newCap = oldCap << 1) < MAXIMUM_CAPACITY &&
                    oldCap >= DEFAULT_INITIAL_CAPACITY) //扩充大小
            newThr = oldThr << 1; // double threshold
    }
    else if (oldThr > 0) // initial capacity was placed in threshold
        newCap = oldThr;
    else {               // zero initial threshold signifies using defaults
        newCap = DEFAULT_INITIAL_CAPACITY;
        newThr = (int)(DEFAULT_LOAD_FACTOR * DEFAULT_INITIAL_CAPACITY);
    }
    if (newThr == 0) {
        float ft = (float)newCap * loadFactor;
        newThr = (newCap < MAXIMUM_CAPACITY && ft < (float)MAXIMUM_CAPACITY ?
                    (int)ft : Integer.MAX_VALUE);
    }
    threshold = newThr;
    @SuppressWarnings({"rawtypes","unchecked"})
        Node<K,V>[] newTab = (Node<K,V>[])new Node[newCap]; // 扩容为新的容量
    table = newTab;
    if (oldTab != null) {
        for (int j = 0; j < oldCap; ++j) {
            Node<K,V> e;
            if ((e = oldTab[j]) != null) {
                oldTab[j] = null;
                if (e.next == null)
                    newTab[e.hash & (newCap - 1)] = e;
                else if (e instanceof TreeNode)
                    ((TreeNode<K,V>)e).split(this, newTab, j, oldCap);
                else { // preserve order 根据新的掩码将这个链表分成两个链表
                    Node<K,V> loHead = null, loTail = null;
                    Node<K,V> hiHead = null, hiTail = null;
                    Node<K,V> next;
                    do {
                        next = e.next;
                        if ((e.hash & oldCap) == 0) {
                            if (loTail == null)
                                loHead = e;
                            else
                                loTail.next = e;
                            loTail = e;
                        }
                        else {
                            if (hiTail == null)
                                hiHead = e;
                            else
                                hiTail.next = e;
                            hiTail = e;
                        }
                    } while ((e = next) != null);
                    if (loTail != null) {
                        loTail.next = null;
                        newTab[j] = loHead;
                    }
                    if (hiTail != null) {
                        hiTail.next = null;
                        newTab[j + oldCap] = hiHead;
                    }
                }
            }
        }
    }
    return newTab;
}

线程安全性

HashMap不是线程安全的，如果有多个线程同时操作这个hashmap，可能会造成死循环，如果有多个线程使用，则应该使用ConcurrentHashMap.

在JDK1.7之前，ConcurrentHashMap使用的是分段锁，也即在ConcurrentHashMap内部预先申请一部分锁，然后计算key的hashcode，根据hashcode返回对应的锁。JDK1.8之后使用的是Hash锁，也即每个hashcode有一个锁。

其put函数变为：

/** Implementation for put and putIfAbsent */
final V putVal(K key, V value, boolean onlyIfAbsent) {
    if (key == null || value == null) throw new NullPointerException();
    int hash = spread(key.hashCode());
    int binCount = 0;
    for (Node<K,V>[] tab = table;;) {
        Node<K,V> f; int n, i, fh;
        if (tab == null || (n = tab.length) == 0) // hashmap为空时初始化table
            tab = initTable();
        // 当插入位置为空时直接插入
        else if ((f = tabAt(tab, i = (n - 1) & hash)) == null) {
            if (casTabAt(tab, i, null,
                            new Node<K,V>(hash, key, value, null)))
                break;                   // no lock when adding to empty bin
        }
        else if ((fh = f.hash) == MOVED)
            tab = helpTransfer(tab, f);
        else {
            V oldVal = null;
            synchronized (f) { // 对插入链表的第一个节点加锁
                if (tabAt(tab, i) == f) {
                    if (fh >= 0) {
                        binCount = 1;
                        for (Node<K,V> e = f;; ++binCount) {
                            K ek;
                            if (e.hash == hash &&
                                ((ek = e.key) == key ||
                                    (ek != null && key.equals(ek)))) {
                                oldVal = e.val;
                                if (!onlyIfAbsent)
                                    e.val = value;
                                break;
                            }
                            Node<K,V> pred = e;
                            if ((e = e.next) == null) {
                                pred.next = new Node<K,V>(hash, key,
                                                            value, null);
                                break;
                            }
                        }
                    }
                    else if (f instanceof TreeBin) {
                        Node<K,V> p;
                        binCount = 2;
                        if ((p = ((TreeBin<K,V>)f).putTreeVal(hash, key,
                                                        value)) != null) {
                            oldVal = p.val;
                            if (!onlyIfAbsent)
                                p.val = value;
                        }
                    }
                }
            }
            if (binCount != 0) {
                if (binCount >= TREEIFY_THRESHOLD)
                    treeifyBin(tab, i);
                if (oldVal != null)
                    return oldVal;
                break;
            }
        }
    }
    addCount(1L, binCount);
    return null;
}

扔鸡蛋问题

发表于 2020-07-29 |

两个鸡蛋，100层楼

假设现在有两个鸡蛋，在某一层楼会碎掉，用最少的次数测出会在第几层楼摔碎。

假设在最坏情况下需要扔x次，那么第一次定义是在第x层楼扔的，如果在该层碎了，那么从第1层遍历到第x层，共测试x次；如果没碎，那么下一次在第\(x+x-1\)层测试。在第\(x+x-1\)层，如果碎了，那么从第\(x+1\)层遍历到\(x+x-1\)层，共测试的次数为\((x-1)-1+2=x\)次,如果没碎，那么继续在\(x+x-1+x-2\)处测试，以此类推。

那么在最坏的情况下，测试到x减到1的时候，可以检测出的楼层数为:

\[ x + (x-1) + (x-2) + (x-3) + \dots 1 = \frac{x(x+1)}{2} \]

为了检测出100层楼中导致鸡蛋碎了的楼层，则需满足：

\[ \frac{x(x+1)}{2} \geq 100 \\ x \approx 14 \]

有了这个公式之后，我们就可以把这个问题推广到，当有N层楼的时候：

\[ \frac{x(x+1)}{2} \geq N \]

K个鸡蛋，N层楼

当有K个鸡蛋N层楼时，就不能用2个鸡蛋的思路去做，但可以很容易想到一种递归的思路。

假设在第i层楼扔一个鸡蛋，那么会有两种情况：

  1. 碎了，则问题转化为，使用K-1个鸡蛋，测试i-1层楼需要的测试次数
  2. 没碎，那么问题转化为，使用K个鸡蛋，测试N-i层楼需要的测试次数

而在第i层楼测试最坏的情况（测试次数最多）就是二者的最大值。那么要找到N层楼中，最坏情况下扔的次数最小的一层楼，就需要从第一层开始计算到第N层楼，求出最小的一层的次数，即为答案。

那么用数学来表示就是，令W(K,N)为使用K个鸡蛋，测试N层楼，需要最少的次数。则：

\[ W(K, N) = \text{min}_{1\leq X \leq N}(\text{max}(W(K-1, X-1), W(K, N-X))) + 1 \]

边界条件为：

  • \(W(K, 0) = 0\),0层台阶不需要测试
  • \(W(K,1) = 1\)，1层台阶只需要测试一次
  • \(W(1, X) = X\)，在只有1个鸡蛋的情况下，只能依次遍历，需要的次数为X

动态规划+二分查找

观察

\[ W(K, N) = \text{min}_{1\leq X \leq N}(\text{max}(W(K-1, X-1), W(K, N-X))) + 1 \]

中max函数部分，\(W(K-1,X-1)\)随X的增加，测试次数也增加，因此是增函数；而\(W(K, N-X)\)随X的增加，测试次数减少，因此是减函数，则二者函数图像如下图所示：

红色部分为max函数，因此最低点的位置就是两个函数相等的位置，因此可以通过二分法逐步逼近最小值。

进一步缩小查询范围

使用动态规划+二分查找的算法可以将最内层循环的时间复杂度降为\(O(NlogN)\),其中每次进行二分查找都是在范围\([0, N]\)之间查找的，继续分析函数属性可以看到：

\(W(K-1,X-1)\)随x增加而增加，但与N无关；而\(W(K, N-X)\)随X的增加而减小，随N的增加而增加，也即二者的关系为：

当测试不同的楼高时，就不必在\([0,j]\)之间使用二分查找，只需从上次最优的j开始找即可。

优化存储空间

可以看到，每次计算最优值时，当前的值仅与i-1个鸡蛋时的值有关，因此可以仅用两个数组即可。

class Solution:
    def superEggDrop(self, K: int, N: int) -> int:
        dp1 = range(N+1)
        for i in range(2, K+1):
            k = 1
            dp = [0, 1] + [float('inf')]*(N-1)
            # j是当前测试的楼层
            for j in range(2, N+1):
                while k < j+1 and dp1[k-1] < dp[j-k]:
                    k += 1
                dp[j] = dp1[k-1] + 1
            dp1 = dp
        return dp1[-1]

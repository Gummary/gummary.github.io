2.1.2

这一节讲了数据抽象的基本思想，也就是为一个特定的数据结构定义一些特定的过程（比如加减乘除），所有对该数据结构的操作都基于这些过程进行。然后讲了抽象屏蔽，我的理解就是分层，将一个程序分层，每一层为上层提供一些过程，同时使用下层提供的过程来实现本层的过程。这样做的好处就是可以提高程序的灵活性，便于修改和维护。

2.1.3

从这节开始画风就变了，难度加大。

这一节的题目直接就是一个问题，数据意味着什么，当时这个标题的时候还真是蒙了一下，当时想的是数据不就是数或数的集合吗。然后继续看书，书上给了一个数据的一般定义：

In general, we can think of data as defined by some collection of selectors and constructors, together with specified conditions that these procedures must fulfill in order to be a valid representation

当时看这段话还没什么感觉，看了书上讲的序对的实现才明白。

序对就是将两个东西组合起来，然后还可以随意取出其中的任意一个。也就是说任何能够满足这两个条件的过程都可以实现序对，然后书上就给了一个例子来实现序对。这个例子没有使用任何的数，仅仅利用过程就实现了序对，并且没有依赖传入的是什么，这个过程都可以将其组合成一个序对。

这个过程可以作为模糊数据与过程界限的一个典型，对于上层来说，cons返回的应该就是一个数对，但是实际上它是一个过程。

在看着道题之前先抛弃以前对数的所有概念，同时问下自己0,1,2,3.....代表着什么，然后再来看这个题。

首先，0,1,2,3...只是人们用来计数的方式，在古代没有这些阿拉伯数字的时候，有结绳计数，中国古代有算筹，其实这些都是计数系统。换句话说，任何能够表达出0,1,2....这些数的意义的过程都可以组成一个计数系统来代替他们。

那么回到我们这道题，先看一下zero和add-1过程的定义

zero是返回了一个过程，这个过程接受一个参数f，然后又返回了一个过程，第二个过程接受一个参数x，然后什么都不做直接返回x

add-1接受一个参数n，然后返回一个过程，该过程接受一个过程f作为参数，然后又返回一个过程，第二个过程也接受了一个参数x，然后先对f应用n，在对返回值应用x，然后将返回值应用给f后返回。

练习2.9

加减很简单，用两个区间试一下就可以了，但是乘和除开始没看懂，仍是搜了一下题解。

这个题要证明的是乘积或商的宽度是两个乘数或除数宽度的函数，那什么函数。函数就是说输入相同那输出也应该相同。所以我们可以通过两对拥有相同宽度的区间进行判断，看结果是否相同。

例如:

[0, 10] * [0, 2] = [0, 20] (width = 10)

[-5, 5] * [-1, 1] = [-5, 5] (width = 5)

所以说并不相同。

练习2.12

假设有两个区间，中间值分别为x,y，误差百分比为dx,dy，则两个区间可表示为:

[x - xdx, x + xdx] [y - ydy, y + ydy]

两个区间相乘之后的结果为

[xy(1-(dx+dy))+xydxdy,xy(1+(dx+dy))+xydxdy]

由于dx，dy比较小，所以可以忽略，变为：

[xy(1-(dx+dy)),xy(1+(dx+dy))]

得到最后的结论，两区间相乘的结果的误差值可以近似为两区间误差之和

这三个问题其实都是关于区间的一致性（唯一确定一个区间），假设我们有两个数A和B位于以下两个区间中

A = [2,8]
B = [2,8]

A可能是3.782，B可能是5.42。

如果让A/A，那么结果一定是1（假设A不为0），如果A/B，那么结果是未知的，只能说位于区间[0.25,4]中。

但是在我们这个系统当中，没有任何有关一致性的判定，也就是说A/A的结果也位于[0.25,4]当中。

所以每当进行的代数运算中包含区间时，都要小心的对待其中相同的区间，因为这重新引入了不确定性。

练习2.14中Lem是正确的，第一个式子中R1，R2各引入了两遍，第二个只有一遍，所以结果不一致。

练习2.15 Eva说的也是正确的，不确定的因素减少了，所以更好。

练习2.16 出现不同结果的原因就是因为没有对一致性做一个定义。所以如果要设计一个没有缺陷的区间算书包就必须为区间一致性做一个定义。

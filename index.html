问题描述

这里直接推广到给定一个数组，其中只有一个数出现p次，其余数均出现k次。

数全部是由1bit组成的

假设有一个数组，里面只由0和1组成的，下面需要用一个计数器利用位操作统计其中1出现的次数。当这个计数器计数到某一个值，如本题中的k时，就要将计数器清零。假设计数器的二进制表示为\(x_m,\dots,x_1\).该计数器满足：

  1. 初始值为0
  2. 当遇到0时值不变
  3. 遇到1时加1
  4. 计数器的最大值要大于等于k，则\(2^m\geq k\)得\(m \geq \text{log}_2^k\)

而满足第二条性质的位运算符有x|=0,x^=0.暂时还无法判断具体使用哪一个运算符。那么直接看累加的过程。

对于计数器的最低位\(x_1\)来说，遇到0不变；遇到1时，若原来为0，则变为1，若原来为1，则变为0。由此我们可以判断只有异或操作满足这个性质，因此我们用异或更新计数器。对于第二位\(x_2\)的变化，只有当第一位为1并遇到1，其值才发生改变，因此x2=x2^(x1&i),依次类推，xm=xm^(xm-1 & ... & x1 & i)。

有了计数器的更新规则后，下面限制计数器值的范围。因为现在计数器的范围是在\(0-(2^m-1)\),为了将计数器限制在k内，我们需要设计一个mask，使得当计数器的值为k时，mask的值为0，其余为1.我们可以判断k的二进制位和计数器的二进制位是否完全相同求得mask。

令\(\text{mask}=\sim (y_m \& \dots \& y_1)\).其中，若\(k_i=1\),则\(y_i=x_i\)；若\(k_i=0\),则\(y_i=~x_i\).这样就可以保证，只要有一位\(k_i\)与\(x_i\)不相同，对应位的\(y_i=0\)，则\(\text{mask}=1\),当完全相同时，mask=1.

最短路径问题

现在有n个点，有m条双向路径，有k个传送门，传送门表示两个路径之间的距离为0，传送门为单向的，求从0到n之间的最短路径。

利用Dijkstra算法求最短路即可。

Dijkstra算法是，假设求P点到其他所有点之间的最短路径，则维护一个数组dist，值为p点到每个点的最短距离；然后维护一个set，内容为已经求出最短距离的点集。则具体步骤为：

  1. 初始化dist数组，dist[p] = 0，将所有与p相邻的点加入到一个队列中
  2. 从队列中选出与p距离最小的点q加入最短点集中，将与q相邻的点加入到队列里，更新所有不在最短点集中的点的距离
  3. 重复2，直到队列为0

单调递增栈与单调递减栈

发表于 2020-08-13 |

单调递增栈和单调递减栈是栈的一种应用，单调递增和递减栈中元素按单调递增和递减排列。以单调递增栈为例，说明单调递增栈的数据更新方式为：

  1. 若当前元素比栈顶元素大，则入栈
  2. 若当前元素比栈顶元素小，则将栈顶元素出栈，直到栈顶元素小于当前元素。

通过这种方式可以得到每个元素左起第一个比当前元素小的元素位置。对于单调递减栈，可以得到每个元素左起第一个比当前元素大的元素位置。

单调递减栈的模板为：

\[ for i, x in enumerate(nums): while x < stack[-1]: stack.pop() stack.push(x) \]

递减栈例题

给定一些矩形，求矩形之间空隙的面积和。

利用单调递减栈，每当遇到一个比栈顶元素大的直方图，就计算与之前最小的直方图构成的间隙面积。

class Solution:
    def trap(self, height: List[int]) -> int:
        stack = []
        ans = 0
        for i in range(len(height)):
            while len(stack) >0 and height[i] > height[stack[-1]]:
                current = stack.pop()
                if len(stack) == 0:
                    break
                h = min(height[i], height[stack[-1]]) - height[current]
                w = i - stack[-1] -1
                ans += h * w
            stack.append(i)
        return ans

01背包问题

问题描述

假设有N个物体，第i个物体的重量为w[i],价值为c[i],背包容量为C,那么将哪些物体放入到背包中，可以在不超过背包容量的情况下使背包所装的价值最大

首先定义状态f[i][v]，其中\(i\in \{1,\dots N\}, v\in \{1, \dots C\}\),则f[i][v]的含义是取前i件物品，容量恰好为v的情况下的最大价值。

则状态转移方程为：

\[ f[i][v] = \text{max}(f[i-1][v], f[i-1][v-w[i]]+c[i]) \]

也即每个物体都有两个状态(01背包的由来)，拿或者不拿。那么：

  • 如果不拿，那么最大收益为取前i-1个物体，容量恰好为v的价值
  • 如果拿，那么最大收益为前i-1个物体，容量恰好为v-w[i]的价值加上当前物体的价值。

注意f[i][v]是有意义的前提是，前i个物体，恰好能组成一个重量为v的背包，因此最终的答案不是f[N][C],因为可能没法恰好组成重量为C的背包，而是\(f[N][0\dots C]\)的最大值。如果移除恰好，那么状态转移方程为：

\[ f[i][v] = \text{max}(f[i-1][v], f[i-1][v-w[i]]+c[i], f[i-1][v-1]) \]

这个算法的时间复杂度为\(O(N*C)\),N为物体数量，C为背包容量，时间复杂度为\(O(N*C)\)，观察这个可以发现， 每次判断时仅使用了dp数组每一行的上一行，因此可以将空间复杂度降低到\(O(C)\)

注意内层循环必须是从最大的背包容量开始迭代，这样可以保证推导dp[v]中使用的dp[v-w[i]]是dp[i-1][v-w[i]]的状态。

多重背包问题

问题描述

给定N个物品，第i件物品的重量为\(w[i]\),价值为\(v[i]\),数量为\(n[i]\),背包容量为C，在不超过背包容量的情况下取得最大的价值。

题目分析

本题与完全背包类似，但是每个物品的数量有限制，与完全背包一样，可以将每个物品拆分成n[i]个，将该问题转为01背包问题，此时时间复杂度为\(O(NV\sum n_i)\)，状态转移方程为：

\[ f[i][v] = \text{max}(f[i-1][v-k\times w_i),\quad 0 \leq k \leq n_i \]

另一种做法是将每个物品的数量用二进制表示，也即每个\(n_i\)可表示成\(1,2, 4, \dots, 2^x, n_i-(2^{x+1}-1)\).其中，\(1,\dots,2^x\)可表示\([1,2^{x+1}-1]\)内的所有数，而\(n_i-(2^{x+1}-1)\) 与\(1,\dots,2^x\)可表示\(2^{x+1},n_i\)之间的所有数。此时，算法时间复杂度为\(O(NV\sum log(n_i))\)

依赖背包问题

问题描述

如果要拿物品i那么必须拿物品j，称这种问题为有依赖的背包问题。为了简化问题，假设存在不依赖其他物品但被其他物品依赖的物品，并且每个有依赖的物品仅会依赖一个物品。

我们将不依赖其他物品的物品称作主件，依赖其他物品的称作附件，那么可以根据主件和附件的关系将所有物品划分为多个组。那么情况变为要么不拿某个主件，要么只拿某个主件，要么拿某个主件并拿一个附件，要么拿一个主件并拿两个附件，依次类推。所以可以将所有物品划分成不同的组，但是每组内的物品可以随意选择，每个组内共有\(2^k+1\)中组合方式。而对于每个组内的每个附件的状态是要么拿要么不拿，很像01背包问题，因此我们可以先在组内做一次01背包，得到\(C-W_k+1\)个代价，其中\(W_k\)为主件k的代价，费用为c的物品的价值为\(f[c-W_k]+V_k\).这些代价之间是由该组内的物品通过不同组合方式组成的，因此彼此之间是互斥的，此时可以直接应用分组背包问题的解决思路解决。

贪心算法题汇编

发表于 2020-08-01 |

贪心算法介绍

贪心算法（英語：greedy algorithm），又称贪婪算法，是一种在每一步选择中都采取在当前状态下最好或最优（即最有利）的选择，从而希望导致结果是最好或最优的算法。比如在旅行推销员问题中，如果旅行员每次都选择最近的城市，那这就是一种贪心算法。贪心算法在有最优子结构的问题中尤为有效。最优子结构的意思是局部最优解能决定全局最优解。简单地说，问题能够分解成子问题来解决，子问题的最优解能递推到最终问题的最优解。贪心算法与动态规划的不同在于它对每个子问题的解决方案都做出选择，不能回退。动态规划则会保存以前的运算结果，并根据以前的结果对当前进行选择，有回退功能。贪心法可以解决一些最优化问题，如：求图中的最小生成树、求哈夫曼编码……对于其他问题，贪心法一般不能得到我们所要求的答案。一旦一个问题可以通过贪心法来解决，那么贪心法一般是解决这个问题的最好办法。由于贪心法的高效性以及其所求得的答案比较接近最优结果，贪心法也可以用作辅助算法或者直接解决一些要求结果不特别精确的问题。

以上摘自维基百科，但是这些概念性的知识并不能帮助我们选择解决实际的问题，而实践是认识的基础、认识的来源、认识发展的动力，因此本文选一些代表性的问题及解法，尽可能的说明白贪心算法在每个问题中的应用。

求重叠区间的最大数量

题目出处

猿辅导笔试

题目描述

给定一些区间[start, end]，求重叠区间的最大数量

题目分析

首先对区间进行排序，先根据起始点排序，若起始点相同则根据终点排序。遍历排序号的区间，用一个最小堆保存当前区间的最小终点。

每次遍历比较最小end与当前区间的起点，

  • 如果起点比最小end小，说明与堆中所有的区间都有交集，入堆
  • 如果起点比最小end大，说明与最小end所在的区间没有交集，可以把当前区间与最小end的区间合并，也即从堆中取出最小end，将当前end加入。可以合并的原因是：
    • 区间已经排好序了，所以之后不可能有区间位于二者之间
    • 这两个区间与堆中的其他区间都重叠，没有增加最大重叠的数量

求非重叠区间的最大数量

题目出处

Leetcode453

题目描述

给定一些区间，求需要移除区间的最小数量，使区间之间不重叠。

题目分析

对区间按照终点进行排序，从最小的重点开始，移除与最小终点重叠的区间，若不冲突则更新最小终点。

这里应用贪心思想的位置在于，尽可能选择终点较小的区间，贪心有效的点在于：

  1. 选择end较小的点，如果发生冲突了，那么因为end较小，所以会有更多的时间选择其他区间。
class Solution:
    def eraseOverlapIntervals(self, intervals: List[List[int]]) -> int:
        if intervals == []:
            return 0
        intervals = sorted(intervals, key=lambda x:x[1])
        current_min = intervals[0][1]
        result = 0
        for interval in intervals[1:]:
            if interval[0] < current_min:
                result += 1
            else:
                current_min = interval[1]
        return result

线程安全性

定义

对线程安全性定义：当多个线程访问某个类时，不管运行时环境采用何种调度方式、线程以何种方式交替执行，主调函数在不采用任何同步措施时，这个类都能表现出正确的结果和行为。

对于不包含任何成员变量，不含对其他类中成员变量引用的类称为无状态类，无状态类一定是线程安全的。

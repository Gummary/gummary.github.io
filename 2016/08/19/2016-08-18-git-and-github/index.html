Git简介

Git是一个分布式版本控制系统，版本控制系统可以获得文件改动的信息，并且可以随时改变当前的软件版本。当然，git的作用还远远不止这些，在接下来会依次介绍git的主要功能。

撤销对文件的修改

这部分涉及到了git的工作区与暂存区的概念，工作区就是你当前写代码的目录，而暂存区位于.git文件夹下， 你每次add之后都将代码提交到了暂存区中，最后commit的时候是将暂存区的所有修改提交到了仓库中。详细的讲解可以查看廖老师的教程，这里就不详细说了。

在add之前撤销

如果你修改一个文件之后还没有add，那么修改仅仅存在于工作区，此时想要回退只能手动删除刚才修改的部分，然后通过git status可以看到文件已经被修改了，此时你可以通过git checkout -- file来丢弃工作区的修改。

在add之后commit之前撤销

如果文件修改完了并add之后，修改就存在与暂存区，这时想要撤销修改，就要使用git reset HEAD file和git checkout -- file来完成。

在commit之后

如果已经commit了，那修改就已经提交到仓库了，就只能使用git reset HEAD^命令来回到上一个版本了

Github

简介

Github与Git本质上是一样的，Github只是一个远程的仓库，你可以将你本地仓库推送的Github上，这样所有人都能看到并下载你的代码，你也可以在Github上寻找其他优秀的代码，然后下载修改。

多人协作的工作模式

节选自廖老师的教程

  1. 首先，可以试图用git push origin branch-name推送自己的修改；

  2. 如果推送失败，则因为远程分支比你的本地更新，需要先用git pull试图合并；

  3. 如果合并有冲突，则解决冲突，并在本地提交；

  4. 没有冲突或者解决掉冲突后，再用git push origin branch-name推送就能成功！

如果git pull提示“no tracking information”，则说明本地分支和远程分支的链接关系没有创建，用命令git branch --set-upstream branch-name origin/branch-name。
